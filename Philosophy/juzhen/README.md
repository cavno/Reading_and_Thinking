# 公理组合矩阵 · Axiom Combinatorics

把四条逻辑律（同一律、矛盾律、排中律、充足理由律）当作**一个集合 / 矩阵**：每条可取
**持守 · 极致 · 松动 · 否弃 · 限定 · 悬置 · 不涉** 之一。于是每一种**排列组合**（共 **7⁴ = 2401** 种）
就是一个「公理构型」，**对应一种哲学倾向（流派 / 哲学家 / 主义）**。

这一视角**打破哲学的历史发展线**：不再按年代排队，而是按构型聚类——
相隔两千余年的普罗泰戈拉、克尔凯郭尔、詹姆斯，因同为「守同一·松矛盾·松排中·松理由」而归为同一倾向；
高尔吉亚、尼采、萨特亦然。

> 与《逻各斯原本》《逻辑思脉地图》同源、同一套数据（101 位哲学家），但换成**组合 / 矩阵**的眼光。
> 两个文件：`index.html` + `data.js`，**可直接部署到 GitHub Pages**。

---

## 怎么用

* **公理组合矩阵（上方 4×7）**：每一行是一条律，每一列是一个取值。**点一个格子**即把该律设为此值；点「任意」放开该条（不约束）。
* **星图（左）**：把所有**已被占据**的构型排布在二维公理空间——
  **横轴＝形式逻辑的硬度**（右＝经典严格，左＝消解），**纵轴＝充足理由的承诺**（上＝极致 / 必然，下＝否弃 / 偶然）。
  圆点＝一个构型，**大小 ∝ 同型人数**，**颜色＝矛盾×排中所落的逻辑家族**（经典 / 直觉主义系 / 次协调系 / FDE·解构系）。
* **联动**：在矩阵里设定取值 → 星图高亮匹配的构型、右侧列出它们；点任一构型（星图圆点或卡片）→ 看该**倾向**及其**同型哲学家**；再点某位姓名 → 看其单独注解与「同型者」。
* **🎲 随机一种**：掷骰落到一个真实存在的构型。**⊞ 浏览全部组合**：列出全部 45 种已占据构型。**↺ 重置**。

在 101 位哲学家中，2401 种组合只有 **45 种被占据**；其余 2356 种是空格——可作为「逻辑上可能、历史上无人」的思想位置来想象。

## 部署到 GitHub Pages

把 **两个文件都放进仓库根目录**（`index.html` 与 `data.js` 必须同目录）：

**网页上传**：新建仓库 → Add file → Upload files 同时拖入 `index.html` 和 `data.js` → 提交 →
Settings → Pages → Source 选 `Deploy from a branch`，分支 `main`、目录 `/ (root)` → 访问 `https://<用户名>.github.io/<仓库名>/`。

**命令行**：
```bash
mkdir axiom-combinatorics && cd axiom-combinatorics
# 把 index.html 和 data.js 放进来
git init && git add index.html data.js && git commit -m "公理组合矩阵"
git branch -M main && git remote add origin https://github.com/<用户名>/<仓库名>.git
git push -u origin main
```
然后在 Settings → Pages 选 `main / root` 开启。

> ⚠️ `index.html` 通过 `<script src="data.js">` 读取数据，所以**两个文件必须一起部署、同一目录**；缺了 `data.js` 页面会空白。

## 说明
* 字体经 Google Fonts 引用，联网自动加载，离线回退系统衬线。
* **坐标公式**：强度映射 极致+2 / 持守+1 / 限定0 / 悬置−0.6 / 松动−1 / 否弃−2 / 不涉0；横轴＝(同一+矛盾+排中)/3，纵轴＝充足理由。同坐标的构型加了微抖动以便区分。
* 立场归类是对各家主张的**提要式**刻画；个别构型的命名见右侧面板，未命名者按四律配置自动生成描述性标签（如「同守·矛极·排守·理守 倾向」）。

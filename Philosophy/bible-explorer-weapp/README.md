# 圣经互动浏览器 · 微信小程序版

> 由网页版移植而来的微信小程序——以五个维度（结构、时间线、故事、人物、主题）系统呈现旧约与新约 66 卷书。共 144 个可点击条目，每个都内置详细介绍。

## 与网页版的区别

小程序不是浏览器，不能直接运行网页代码。本版本用**相同的数据**重写了一套小程序前端：

| 网页版 | 小程序版 |
|---|---|
| HTML + CSS + JS | WXML + WXSS + JS |
| `innerHTML` 注入详情 | `<rich-text>` 组件渲染详情 |
| `<script src>` 引数据 | `require()` 引入数据模块 |
| CSS `px` | WXSS `rpx`（自适应） |
| Google Fonts | 系统衬线字体（宋体观感） |
| localStorage 存主题 | `wx.setStorageSync` 存主题 |

数据文件（`data/*.js`）与网页版内容完全一致，只是改用 `module.exports` 导出。

## 如何在微信开发者工具中预览

1. 下载并安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 解压本项目
3. 打开开发者工具 → `导入项目`
4. 目录选择解压后的 `bible-explorer-weapp` 文件夹
5. AppID：
   - 若只想本地预览，选择 **「测试号」** 或 **「不使用 AppID（游客模式）」**（`project.config.json` 已预置 `touristappid`，可直接进入游客模式）
   - 若要真机预览或上架，需填入自己的小程序 AppID（在[微信公众平台](https://mp.weixin.qq.com)注册获得）
6. 点击 `导入`，工具会自动编译并在模拟器中显示

### 真机预览

在开发者工具中点击工具栏的 `预览`，用微信扫描生成的二维码即可在手机上查看（需使用已绑定的 AppID，且当前微信号是该小程序的开发者/体验者）。

## 功能

- **五个标签页**：概览 / 时间线 / 故事 / 人物 / 主题，顶部横向切换
- **详情抽屉**：点击任一条目，从右侧滑出详细介绍
- **深浅主题**：点击右上角 ◐ 切换，偏好保存在本地；首次进入会尝试跟随系统
- **原生导航栏联动**：切换主题时，顶部导航栏颜色同步变化

## 字体说明

小程序无法像网页那样加载 Google Fonts。本版本使用系统字体：

- **中文衬线**：iOS 上为「宋体（Songti SC）」，呈现书卷气；Android 各厂商字体不一，会回退到系统默认
- **拉丁字母**：回退到 Georgia / Times New Roman 等系统衬线

若想要统一的拉丁展示字体，可在 `app.js` 的 `onLaunch` 中用 `wx.loadFontFace` 加载一个 HTTPS 上的 woff2 字体文件（注意中文字体体积大，不建议整体加载）。

## ⚠️ 上架与合规须知（重要）

这一点必须如实说明，请务必先评估：

**中国大陆对互联网宗教信息有专门监管**。根据《互联网宗教信息服务管理办法》（2022 年 3 月施行），在互联网上提供宗教信息服务（包括传授、阐释宗教内容等）需要取得**《互联网宗教信息服务许可证》**。

对小程序而言，这意味着：

- 涉及宗教内容的小程序，微信审核时通常要求提供上述许可证，**个人主体基本无法获得**
- 未经许可发布宗教信息存在合规风险

因此，本项目更适合的用途是：

1. **个人学习 / 研究** —— 在开发者工具中本地使用，或仅在自己掌握的小范围内用真机预览（体验者名单可加最多若干人）
2. **海外或非大陆场景** —— 若面向海外用户，限制少很多，网页版（可部署到 Vercel / Cloudflare Pages）通常是更灵活的选择
3. **通过有资质的宗教团体合法渠道** —— 若确有公开发布需求，应经由具备许可证的主体操作

如果你的目的是私人学习或技术研究，本地预览完全没有问题。如果计划公开发布，请先确认主体资质与内容合规。

## 技术要点（供二次开发参考）

- **数据与视图分离**：详情内容存在模块级 `INDEX` 对象中（不进 `data`），点击时才查表并 `setData`，避免初始渲染负担过重
- **rich-text 渲染**：详情 HTML（含 `<strong>` `<br>`）经 `toNodes()` 转为带内联样式的节点串，主题切换时会按当前主题色重新生成
- **主题变量**：浅色变量定义在 `page` 上，深色变量定义在 `.theme-dark` 包裹层，通过 CSS 变量级联实现整体换肤
- **抽屉动画**：用 `visibility` 的 `transition-delay` 技巧实现「即时滑入、延迟隐藏」，无需 JS 定时器
- **背景滚动锁定**：抽屉打开时通过 `<page-meta>` 设置页面 `overflow: hidden`

# 西方哲学史考察 · 前四讲 — 微信小程序部署说明(保姆级)

> 把梅春雷老师《西方哲学史考察十讲》前四讲讲义做成的**原生文字小程序**:
> 一条主轴 + 六条贯穿线 + 四讲全文要义 + 概念谱系 + 张力总表 + 人物长廊。
> 纯本地数据、零网络请求、零图片资源,整包约 130KB,个人主体可直接上线。
> 文件包:`xizhe-shijiang.zip`(解压得文件夹)。

## 0. 这个小程序里有什么

| 页面 | 内容 |
|---|---|
| **首页(脉络中枢)** | 课程主轴链条(人类出现 → … → 科学,深色为本四讲覆盖段)、孔德引言、思想脉络入口、四讲卡片、四个专题入口 |
| **讲义页 ×4** | 每讲含:一句话定位、核心之问、承前/启后、正文(节标题/定义卡/卡组/编号要点/对照表/人物卡/强调横幅/承上启下/书目),底部可前后翻讲 |
| **思想脉络** | 主轴七站(可点跳对应讲义)+ 六条贯穿线:恒常的追问 · 连续与转换 · 命运的长河 · Logos 的漂流 · 解释权的转移 · 语言的边界 |
| **谱系与张力** | 双标签:六条概念长河(Logos / 命运 / 一 / 灵魂净化 / 神圣世俗 / 共相)+ 三组张力(原始之分 / 希腊十六对 / 神学五大矛盾) |
| **人物长廊** | 九个时代带、52 位思想者,从巫师到加尔文再到回望者(人类学家与哲学家) |

设计与本系列既有小程序一致:羊皮纸 `#f3ebda` / 墨色 `#1c1916` / 赭红 `#a73e2c` / 爱琴蓝 `#1d4e5f` / 鎏金 `#b8893a`,本作新增神学酒红 `#7e2a38`;每讲一种强调色(一讲赭红、二讲鎏金、三讲爱琴蓝、四讲酒红)。

```
xizhe-shijiang/
├─ app.js / app.json / app.wxss          ← 全局配置与设计系统
├─ project.config.json / project.private.config.json / sitemap.json
├─ data/
│  ├─ lectures.js   ← 四讲全文要义(119 个 section,结构化)
│  ├─ threads.js    ← 主轴 7 站 + 贯穿线 6 条
│  ├─ atlas.js      ← 概念长河 6 条 + 张力 3 组 27 对
│  └─ figures.js    ← 人物 9 带 52 位
└─ pages/
   ├─ index/    ← 首页(脉络中枢)
   ├─ lecture/  ← 讲义页(?id=1..4 数据驱动)
   ├─ thread/   ← 思想脉络
   ├─ atlas/    ← 谱系与张力(?tab=concepts|dual)
   └─ figures/  ← 人物长廊
```

## 2. 最快看到效果(免注册)

1. 安装**微信开发者工具(稳定版)**:
   `https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html`
2. 解压本 zip 得到 `xizhe-shijiang` 文件夹。
3. 打开工具 → 手机扫码登录 → **「+ 导入项目」** → 目录选该文件夹 → AppID 点 **「测试号」** → 导入。
4. 模拟器即可浏览;右上角 **「预览」** 扫码上真机。

## 3. 正式上线

1. **注册账号**:`mp.weixin.qq.com` → 立即注册 → 选「小程序」→ 邮箱激活 → 选主体(个人/企业)→ 完成验证 → 「开发 → 开发管理 → 开发设置」复制 **AppID**。
2. **填 AppID**:开发者工具「详情 → 基本信息」修改 AppID;或直接改 `project.config.json` 里的 `"appid"`。
3. **预览 / 真机调试**确认无误。
4. **上传**(右上角「上传」,填版本号 1.0.0 + 备注)。
5. 后台「管理 → 版本管理」看到开发版本 → 「选为体验版」可先内测。
6. **提交审核**:服务类目建议「教育 / 教育信息服务」或「工具」;功能页填 `pages/index/index`;说明可写「哲学课程讲义阅读,纯本地文字、无网络请求、不收集任何信息」。
7. 审核通过 → **发布** → 获得正式二维码。

## 4. 常见问题与自定义

- **AppID 报错**:先用「测试号」;正式上线前确认 `project.config.json` 的 `appid` 已替换。
- **改字号 / 行距**:全局正文在 `app.wxss` 的 `page`(30rpx / 1.85);讲义正文卡片在 `pages/lecture/lecture.wxss` 的 `.cg-d`、`.num-d` 等。
- **改配色**:全局色值集中在 `app.wxss` 顶部注释与各 wxss 中的十六进制;四讲强调色由 `data/lectures.js` 每讲的 `accent` 字段(terra/gold/aegean/wine)控制。
- **改内容 / 补充要义**:全部文字在 `data/` 四个文件中,均为结构化纯文本;`lectures.js` 的 section 类型有 12 种(h 节标题 / lead 导语 / quote 引言 / def 定义卡 / cards 卡组 / nums 编号要点 / vs 对照表 / flow 链条 / persons 人物卡 / band 强调横幅 / bridge 承上启下 / books 书目),照现有格式增删即可,无需动页面代码。
- **后续六讲怎么加**:在 `data/lectures.js` 数组里按同样结构追加 `id: 5` 起的讲次,首页与翻页导航自动生效;主轴与贯穿线在 `data/threads.js` 中按需补站。
- **个人主体**:原生实现、零网络请求、不用 web-view,不受个人主体限制。
- **基础库**:默认 3.5.5;可在「详情 → 本地设置 → 调试基础库」选较新稳定版。

# 微信小程序部署 · 保姆级全流程

> 项目：**四律 · 逻辑思脉地图（科普勒斯顿全谱系）** —— 由网页版完整移植的原生小程序。
> 文件包：`logos-atlas-miniapp.zip`（解压后是文件夹 `logos-atlas-miniapp/`）。

按下面的步骤一步步来，**从「只想在电脑上看到它跑起来」到「正式上线给别人用」全程都有**。看不懂的地方照着截图位置点就行。

---

## 0. 先认识一下你拿到的东西

解压后文件夹结构（这就是一个标准的微信小程序工程，不用改动也能直接导入）：

```
logos-atlas-miniapp/
├─ app.js / app.json / app.wxss      ← 小程序全局入口与配置
├─ project.config.json               ← 工程配置（含 appid 占位）
├─ sitemap.json
└─ pages/index/
   ├─ index.wxml   ← 页面结构（相当于 HTML）
   ├─ index.wxss   ← 页面样式（相当于 CSS）
   ├─ index.js     ← 页面逻辑（开关筛选 / 四象限 / 松绑史 / 详情）
   └─ data.js      ← 101 位哲学家 + 26 时代带 + 谱系数据
```

**移植说明（一处合理改动）**：网页版用 SVG 曲线连接各思脉；手机端整张图高达约 5700px，做满屏 Canvas 会超出手机纹理上限，所以小程序里**把「连线」改成了「点亮相连的节点」**——点任一哲学家，它的「承自/影响」会一起高亮（金色边框），详情面板里也有可点的关系列表。其余（四律开关三档、绿/红/黄＋镶环立场色、矛盾×排中四象限、松绑史逐幕播放、筛选取交集）与网页版完全一致。

## 1. 最快路径：先在电脑上把它跑起来（无需注册账号，3 步）

如果你只是想先看到效果，**不需要任何账号**：

1. **下载微信开发者工具（稳定版 Stable）**
   打开官网下载页：`https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html`
   选对应系统（Windows 64 / macOS / macOS Apple Silicon）下载并安装。

2. **解压项目**：把 `logos-atlas-miniapp.zip` 解压，得到文件夹 `logos-atlas-miniapp/`。

3. **导入并预览**：
   - 打开微信开发者工具，用**手机微信扫码登录**（开发者工具本身要登录，但这一步用你自己的微信扫码即可，和注册小程序无关）。
   - 点左侧 **「+ 导入项目」**（或菜单「项目 → 导入项目」）。
   - **目录**：选刚解压的 `logos-atlas-miniapp/` 文件夹。
   - **AppID**：先点 **「测试号」**（也叫「使用测试号 / touristappid」）——这样不用注册就能预览。
   - 点 **「导入」**。

   进去后，左边「模拟器」就会显示这个小程序在跑。顶部四个开关（守/中/弃）、地图、底部抽屉面板都能点。
   想在真手机上看：点工具右上角 **「预览」**，用微信扫弹出的二维码即可（测试号下，预览仅限你本人微信）。

> ✅ 到这里你已经能看到完整效果了。**下面是要正式上线（让别人也能用）才需要做的事。**

---

## 2. 注册一个小程序账号（拿到正式 AppID）

正式发布必须有自己的小程序账号与 AppID。

1. 浏览器打开 **`https://mp.weixin.qq.com`**，点右上角 **「立即注册」**。
2. 账号类型选 **「小程序」**。
3. 填写一个**没有注册过公众平台**的邮箱 → 设密码 → 邮箱收激活邮件 → 点激活链接。
4. **选择主体类型**：
   - **个人**：免费、最快，个人开发者选这个即可（用身份证 + 本人微信扫码验证）。
   - **企业/个体工商户**：需要营业执照、对公验证（或微信认证 ¥30/年）。
   - 政府/媒体/其他组织：按提示走。
5. 按提示填资料、完成主体验证。提交后即可进入小程序后台。
6. 在后台左侧菜单 **「开发 → 开发管理 → 开发设置」**，找到 **「AppID(小程序ID)」**，复制保存——形如 `wx1234567890abcdef`。

> ⚠️ **个人主体**有两点要知道：①不能使用 `web-view` 打开外部网页（所以本项目做成了**原生**小程序，正好不受影响）；②部分服务类目（支付、医疗等）受限——本项目是「工具/教育」性质，不涉及这些。

---

## 3. 把正式 AppID 填进工具

有了正式 AppID 后，让工程用它：

**方式 A（推荐，工具里改）**：
- 打开微信开发者工具，点顶部项目名右侧 **「详情 → 基本信息」**，把 **AppID** 改成你的正式 AppID，确认。

**方式 B（直接改文件）**：
- 用记事本打开 `project.config.json`，把这一行
  `"appid": "touristappid",`
  改成
  `"appid": "你的AppID",`
  保存，回到工具里它会自动刷新。

> 注：`project.config.json` 里 `"urlCheck": false` 已为你设好（本项目不请求任何外部网络，无需配置服务器域名）。

## 4. 真机调试 / 预览（强烈建议上线前做）

- **模拟器**：导入后左侧即时可见，改代码自动热重载。
- **预览**：工具右上 **「预览」** → 生成二维码 → 手机微信扫 → 在手机上体验真实效果（注意：地图很长，上下滑动浏览；用顶部开关筛选或「松绑史」播放更省事）。
- **真机调试**：右上 **「真机调试」** → 扫码 → 可在手机上边操作边看日志，排查问题。

常看的几处是否正常：
1. 顶部四个开关点「守/中/弃」，地图相应节点变亮/变暗；
2. 底部抽屉「现代逻辑四象限」点格子，对应族群高亮；
3. 点任一哲学家：底部出现其四律签名（绿/红/黄＋镶环）、注、谱系；其相连节点变金色边框；
4. 点「▶ 松绑史」→ 底部出现播放控制，逐幕高亮各时代。

## 5. 上传代码到微信后台

1. 在微信开发者工具右上角点 **「上传」**。
2. 弹窗里填：
   - **版本号**：如 `1.0.0`（自定，便于管理）。
   - **项目备注**：如「逻辑思脉地图首个版本」。
3. 点「上传」，等待编译上传完成。
4. 上传成功后，去 **小程序后台（mp.weixin.qq.com）→ 左侧「管理 → 版本管理」**，在 **「开发版本」** 区域就能看到你刚传上去的这一版。

---

## 6. 设为体验版，先内测（可选但推荐）

- 在「版本管理 → 开发版本」里，点你那一版的 **「选为体验版」**。
- 设为体验版后，在「版本管理」上方可以拿到 **体验版二维码**，并可在 **「成员管理 → 体验成员」** 里添加最多若干个体验者的微信号。
- 体验成员扫码即可在自己手机上试用（无需审核）。确认没问题再去提交审核。

---

## 7. 提交审核

1. 后台 **「版本管理 → 开发版本」**，点你那一版的 **「提交审核」**。
2. 第一次提交前，平台会要求先补全信息：
   - **服务类目**：本项目建议选 **「教育 → 教育信息服务」** 或 **「工具」**（具体可选项以后台为准；选与「知识/教育/工具」相关的类目）。提交后类目可能需要资质或简单说明，按提示填。
   - **小程序名称 / 头像 / 简介**：在「设置 → 基本设置」里先填好（名称需唯一、审核）。
3. 提交审核表单里：
   - **功能页面**：填 `pages/index/index`。
   - **测试账号**：本项目无需登录，留空或注明「无需登录，打开即用」。
   - **审核备注**：可写「一个哲学史可视化工具：把四条逻辑规律当作可选公理，展示从古希腊到当代约百位哲学家的立场与谱系。纯本地数据，无网络请求、无用户信息收集。」
4. 提交。审核一般 **数小时到 1～2 个工作日**。结果会站内信/邮件通知。

---

## 8. 发布上线

- 审核**通过**后，回到 **「版本管理 → 审核版本」**，点 **「发布」**（可选「全量发布」）。
- 发布成功后，在 **「设置 → 基本设置」** 底部可获取 **小程序码 / 正式二维码**，分享给任何人，微信扫码即可使用。也可在微信「搜一搜」搜到你的小程序名称。

> 🎉 到此就正式上线了。后续要更新：改代码 → 上传新版本号 → 提交审核 → 发布，循环即可。

## 9. 常见问题（FAQ）

**Q1：导入时报「AppID 不合法」？**
用「测试号」先导入；或确认 `project.config.json` 里 `appid` 已填成你自己的（不要带引号外的空格）。

**Q2：地图太长、滑动卡？**
属正常——101 个节点本就很长。优先用顶部开关**筛选**（点亮符合某配置的哲学家、其余淡出）或「**松绑史**」逐幕跳转，而不是一路手滑。

**Q3：能不能直接把网页版用 `web-view` 套进来就好？**
不建议：① **个人主体小程序不支持 `web-view`**；② `web-view` 要求业务域名已 **ICP 备案** 且能上传校验文件，GitHub Pages 域名做不到；③纯网页打包易被审核驳回。所以这里做成了**原生**小程序，兼容个人主体、无需域名。

**Q4：审核被拒怎么办？**
按拒绝原因改（多为「类目与内容不符」「简介/名称问题」「缺少必要页面」）。本项目无登录、无收集信息、无敏感内容，通常补好类目与简介即可。改完重新「提交审核」。

**Q5：想改标题/配色/内容？**
- 标题：`app.json` 与 `pages/index/index.json` 的 `navigationBarTitleText`。
- 立场颜色：`pages/index/data.js` 里 `STANCE` 的 `pip` 值；或 `index.js` 的 `pipStyle`。
- 增删哲学家：编辑 `pages/index/data.js` 的 `C`（节点）与 `BANDS`（时代带）、并在节点 `lin` 里加谱系边。改完工具会自动重编译。

**Q6：基础库版本？**
工程默认 `libVersion: 3.5.5`。若你的工具版本不同，在工具「详情 → 本地设置 → 调试基础库」选一个较新的稳定版本即可（建议 ≥ 2.9.0，以支持新版组件特性）。
